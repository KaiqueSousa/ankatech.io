Sistemas de Controle de Versões

Podemos dizer que um Sistema de Controle de versão, é um software onde podemos gerenciar as versões no desenvolvimento de um documento qualquer, nele é possível ver o histórico e desenvolvimento dos documentos e também dos códigos-fontes.

Esse tipo de Sistema é usado por empresas de desenvolvimento de software e de tecnologia como também é muito usado em desenvolvimento de software livres, além de serem usados para projetos pessoais pequenos assim também como grandes projetos comerciais.

Alguns exemplos são Subversion, Git, Mercurial entre vários outros:

Subversion

Subversion ou Apache Subversion como também é conhecido é um sistema de controle de versão desenhado especificamente para ser um substituto moderno do Sistema de versões Concorrentes (CVS do inglês Concurrent Version System) que ter algumas limitações.

Git

Git é uma ferramenta livre e código aberto, é um sistema distribuído de controle de versão projetado para lidar com tudo, desde pequenas a grandes projetos com rapidez e eficiência. Git é um sistema fácil de aprender a mexer além de ter uma pegada pequena com desempenho rápido. Ele supera ferramentas de SCM como Subversion, CVC, Perforce, ClearCase com recursos como a ramificações locais, convenientes áreas de teste, e vários fluxos de trabalho.

Mercurial

Mercurial é uma ferramenta multiplataforma de controle de controle de versão distribuído para desenvolvedores de software. O sistema é implementado principalmente em Python, porém o utilitário binário diff foi escrito em C.

Quais as principais vantagens de usar um Sistema de Controle de Versão:

Trabalho em Equipe: Usando algum desses Sistemas é permitido que diversas pessoas possam trabalhar sobre um mesmo conjunto de documentos e consequentemente diminuindo o desgaste provocado por problemas de conflitos com edições.

Controle do histórico : Com o mesmo também se tem a facilidade em desfazer, analisar o histórico do desenvolvimento, como também facilidade no resgate de versões mais antigas e estáveis. E dependendo so sistema a ser usado é possível analisar as alterações com detalhes, desde a primeira versão até a última.

Marcação e resgate de versões estáveis: Dependendo do sistema usado, é permitido marca onde é que o documento estava com uma versão estável, podendo ser facilmente resgatado no futuro.

Ramificação do projeto: Muitos dos sistemas que existem possibilita a divisão do projeto em várias linhas de desenvolvimento, que podem ser trabalhadas paralelamente, sem que uma interfira na outra.
